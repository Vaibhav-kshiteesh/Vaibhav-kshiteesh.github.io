Student Performance Prediction Model

Academic Project  ·  Python · Scikit-learn · Pandas · Seaborn

A complete end-to-end machine learning pipeline that predicts final academic grades from demographic, behavioral, and socioeconomic features. The project covers every stage of the ML lifecycle — from raw data ingestion and exploratory analysis through preprocessing, feature engineering, multi-model comparison, hyperparameter tuning, cross-validation, and permutation-based feature importance — achieving an R² of 0.88 on held-out test data.

  • Loaded and audited a 649-student dataset (mirrors UCI Student Performance) with demographic, behavioral, and parental education features; profiled ~5% missing values.
  • Built a preprocessing pipeline: median imputation, binary encoding for yes/no variables, and label encoding for multi-class categoricals using Scikit-learn.
  • Engineered 5 derived features: avg_grade, grade_trend, parent_edu_avg, alcohol_index, and study_fail_ratio to capture non-linear behavioral interactions.
  • Benchmarked 5 regressors (Linear, Ridge, Lasso, Random Forest, Gradient Boosting); used GridSearchCV (5-fold) to tune the best performer.
  • Validated with 10-fold cross-validation (R² = 0.88 ± 0.03); identified avg_grade as the dominant predictor via permutation importance.

SQL-Based Library Management System

Mini Project  ·  MySQL · DBMS · Normalization

A fully normalized relational database system that manages the core operations of an academic library — cataloguing books, tracking student memberships, recording borrowing history, and enforcing return deadlines. Built to demonstrate rigorous DBMS design principles: normalization up to 3NF, referential integrity, and query optimization through stored procedures and indexed views.

  • Designed an ER diagram and relational schema covering Books, Members, Transactions, Fines, and Staff entities with all relationships and cardinalities documented.
  • Normalized tables to Third Normal Form (3NF) — eliminating transitive and partial dependencies to ensure data integrity and reduce redundancy.
  • Implemented multi-table joins, correlated subqueries, aggregate functions (GROUP BY, HAVING), and referential constraints (ON DELETE CASCADE) to support core workflows.
  • Wrote stored procedures for frequent operations (borrow, return, fine calculation) and views to pre-aggregate overdue records, cutting query complexity for repeat use-cases.
